# 极速吃透：树 数据结构 + 核心算法
## 一、树是什么（一句话）
**树是非线性数据结构**，元素一对多，层级关系。
和线性结构（数组/链表/栈/队列）对立。

---
## 二、基础术语（必背，极简）
- **根节点**：最顶层，唯一，无父节点
- **父/子节点**：上层为父，下层为子
- **叶子节点**：没有子节点
- **深度**：当前节点到根的层数
- **高度**：当前节点到最远叶子的层数
- **度**：节点拥有的子节点个数

---
## 三、最常用：二叉树
### 1. 定义
每个节点**最多2个子节点**：左孩子、右孩子。

### 2. 二叉树分类
1. **普通二叉树**：无限制
2. **满二叉树**：每一层全部长满
3. **完全二叉树**：最后一层从左往右填
4. **二叉搜索树 BST**
   左子树所有值 < 根 < 右子树所有值
   ✅ 高效查找、插入、删除
5. **平衡二叉树(AVL/红黑树)**：防止BST退化成链表

---
## 四、二叉树节点结构（Java 标准模板）
```java
class TreeNode {
    int val;
    TreeNode left;
    TreeNode right;
    TreeNode(int x) { val = x; }
}
```

---
## 五、二叉树核心：四种遍历（必考）
### 记忆口诀
- 前序：**根 → 左 → 右**
- 中序：**左 → 根 → 右**（BST中序=有序数组）
- 后序：**左 → 右 → 根**
- 层序：从上到下、从左到右（借助队列）

### 递归代码（最简）
```java
// 前序
void preOrder(TreeNode root){
    if(root == null) return;
    System.out.print(root.val);
    preOrder(root.left);
    preOrder(root.right);
}

// 中序
void inOrder(TreeNode root){
    if(root == null) return;
    inOrder(root.left);
    System.out.print(root.val);
    inOrder(root.right);
}

// 后序
void postOrder(TreeNode root){
    if(root == null) return;
    postOrder(root.left);
    postOrder(root.right);
    System.out.print(root.val);
}
```

### 层序遍历（队列实现）
```java
import java.util.LinkedList;
import java.util.Queue;

void levelOrder(TreeNode root){
    if(root == null) return;
    Queue<TreeNode> q = new LinkedList<>();
    q.offer(root);
    while(!q.isEmpty()){
        TreeNode cur = q.poll();
        System.out.print(cur.val);
        if(cur.left != null) q.offer(cur.left);
        if(cur.right != null) q.offer(cur.right);
    }
}
```

## 六、树 高频经典算法
1. **递归类（核心思想：分治）**
    - 求树的最大深度
    - 求节点个数
    - 求叶子节点个数
    - 翻转二叉树（左右互换）

2. **二叉搜索树 BST 算法**
    - 查找目标值
    - 插入节点
    - 删除节点
    - 中序遍历得到升序序列

3. **路径问题**
    - 二叉树的直径
    - 根到叶子路径和

4. **迭代遍历**
   栈模拟递归，不用递归实现前/中/后序

---
## 七、树  vs  线性结构
- 数组/链表：一对一
- 树：**一对多**
- 栈、队列常用来：迭代遍历树、层序遍历

---
## 八、极简例题：求二叉树最大深度
```java
public int maxDepth(TreeNode root) {
    if(root == null) return 0;
    int left = maxDepth(root.left);
    int right = maxDepth(root.right);
    return Math.max(left, right) + 1;
}
```

## 九、总结速记
1. 树 = 层级、一对多
2. 二叉树每个节点最多两个孩子
3. 三大递归遍历：前、中、后
4. 层序遍历必用队列
5. BST 左小右大，中序有序
6. 树算法核心：**递归分治**

```java
import java.util.Queue;

void bfs(TreeNode root){

    q.offer(root);

        if(cur.right != null) q.offer(cur.right);
    }
}
```

# 极速吃透：B树、B+树、B*树（数据库/文件系统核心）
先抓核心结论：
**B系列树 = 多路平衡查找树**，专门为**磁盘、海量数据、少IO**设计；
二叉树太深、IO次数多，不适合存数据库，所以诞生B树。

---

## 一、前置铺垫：为什么不用二叉搜索树？
1. 二叉树高度太高，数据千万级时深度几十层
2. 磁盘读写按「页」IO，每下一层就要一次磁盘IO
3. IO很慢，必须**减少树的高度**
   ✅ 解决方案：
   **多叉、矮胖、平衡** → B树、B+树

---

## 二、B 树（多路平衡搜索树）
### 1. 核心定义
- 每个节点**可以有多个关键字、多个子节点**
- 严格平衡：所有叶子节点**同一层**
- 有序、分段划分区间

### 2. 关键规则（必背）
1. 阶数 M 阶B树：
   - 节点关键字数：\(\lceil M/2 \rceil-1 \le 关键字数 \le M-1\)
   - 子节点数 = 关键字数 + 1
2. 节点内关键字**从小到大有序**
3. 左边子树 < 当前关键字 < 右边子树
4. 所有叶子节点在同一层，天然平衡

### 3. 特点
- 查找、插入、删除都稳定
- 高度极低，磁盘IO极少
- 数据+索引**都存在所有节点**

---

## 三、B+树（MySQL 索引主力）
### 1. 在B树基础上改造，数据库专用
### 核心规则
1. **非叶子节点只存索引键，不存真实数据**
2. **所有真实数据、记录，全部存在叶子节点**
3. 叶子节点通过**双向链表串联**
4. 非叶子节点仅用来导航、指路

### 2. B+树 超强优势
1. **单层节点存更多索引**，树更矮，IO更少
2. **范围查询极强**：叶子链表直接遍历区间
3. 全表扫描、范围查找不用中序遍历，直接链表走
4. 非叶子更小，缓存命中率更高

### 3. B树 vs B+树
| 对比 | B树 | B+树 |
|------|-----|------|
| 数据存储 | 所有节点存数据 | 仅叶子存数据 |
| 范围查询 | 弱 | 极强（链表） |
| 树高 | 略高 | 更低 |
| 适用场景 | 文件系统 | MySQL、InnoDB索引 |

---

## 四、B*树
B+树的再优化：
1. B+树节点满了才分裂；
2. **B*树优先兄弟节点转移元素，实在放不下再分裂**
3. 空间利用率更高，分裂次数更少
> 了解即可，面试重点只考 B树、B+树

---

## 五、MySQL 为什么用 B+树 做索引？
1. 磁盘IO少：树矮、层数少
2. 范围查询、排序、分页超级友好（叶子链表）
3. 非叶子节点小，能缓存更多索引，内存命中率高
4. 增删稳定平衡，不会退化

## 六、简单理解图示（文字版）
### B树
```
      [20,50]
   /     |      \
[5,10] [30,40] [60,80]
```
每个节点都有数据+索引。

### B+树
```
        [20,50]
     /       \
[10,20]    [50,70]
  ↓          ↓
叶子链表：10→20→30→50→60→70  （全量数据）
```
上层只当路标，数据全在最下层链表。

2. 逐层向下筛选区间
3. 最终落到**叶子节点**取真实数据

```java
import java.util.ArrayList;

class BTreeNode {

        children = new ArrayList<>();

    }

    // 阶数

    }
```

## 十、终极背诵总结
1. **B树**：多叉平衡树，节点存索引+数据，通用平衡树
2. **B+树**：B树改良，非叶子只存索引，叶子链表存全部数据，数据库专属
3. **B*树**：优化空间利用率，减少分裂
4. 核心目的：**降低树高、减少磁盘IO**
5. 面试必问：MySQL 索引为什么选 B+树
